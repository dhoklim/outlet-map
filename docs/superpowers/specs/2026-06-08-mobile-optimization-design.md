# 모바일 최적화 설계 — 콘센트맵

## 목표

스마트폰에서 콘센트맵 웹을 편하게 쓸 수 있도록 `web/index.html`의 CSS와 JS를 수정한다. 백엔드(storage.py, web_server.py, models.py)는 변경하지 않는다.

---

## 수정 범위

`outlet_map/web/index.html` 단일 파일. 기존 데스크톱 레이아웃(880px 초과)은 동결.

---

## 변경 사항 4가지

### 1. 헤더 슬림화 (모바일 전용)

- 현재: 브랜드 로고 + 제목 + 탭 + 편집모드 체크박스 + 검색 입력창 + 버튼 → 모바일에서 2~3줄로 줄바꿈됨
- 변경: 880px 이하에서 로고·서브타이틀 숨김, 검색창을 아이콘(🔍) 버튼으로 축약, 편집모드를 작은 아이콘 토글(✏️)로 교체
- 헤더 높이를 48px로 고정해 지도 영역을 최대화

### 2. 마커 터치 영역 확대

- 현재: `.marker` 버튼이 22×22px dot 크기 그대로 — 터치 실패 잦음
- 변경: 모바일에서 `.marker`의 실제 터치 영역을 `min-width/min-height: 44px`로 설정 (Apple HIG 권장). dot 시각 크기는 유지하고 투명 padding으로 영역만 확장

### 3. 바텀 시트 패널

- 현재 모바일: 지도 위에 패널이 아래 스크롤로 붙어서 평가하려면 스크롤 필요
- 변경:
  - 880px 이하에서 `.panel`을 `position: fixed; bottom: 0; left: 0; right: 0`으로 전환
  - 기본 상태: 핸들(drag bar) + 기기 이름 + 상태 + 별점 요약만 보이는 **collapsed** 높이(~140px)
  - `renderPanel()` 이후 `panel.classList.add('open')` → 평가 폼까지 보이는 **expanded** 높이(~65vh)
  - 핸들 클릭 또는 "평가하기" 버튼 클릭 시 토글
  - 마커 미선택 시 패널 완전 숨김(`bottom: -100%`)
  - 지도 높이: `calc(100vh - 48px)` (헤더 제외 전체)

### 4. 별점 탭 선택 (모바일 전용)

- 현재: `<select>` 드롭다운 — 모바일에서 OS 네이티브 피커로 열려 느리고 어색함
- 변경: 880px 이하에서 `<select>` 숨기고 ★ 5개를 `<button>`으로 렌더링. 탭하면 1~n번째까지 채워짐. 선택 값은 기존 `rating` 변수와 동기화

---

## 데스크톱 영향 없음

- 스타일 변경은 `@media (max-width: 880px)` 블록 안에만 적용
- JS 동작 분기는 `window.matchMedia('(max-width: 880px)').matches`로 판단
- 기존 CSS 변수, 색상, 컴포넌트 구조 유지

---

## 테스트 체크리스트

- [ ] 데스크톱(1280px): 기존 2칸 그리드 그대로 동작
- [ ] 모바일(375px iPhone SE 기준): 지도 전체화면, 헤더 48px
- [ ] 마커 탭 → 바텀 시트 슬라이드업
- [ ] 시트 핸들 탭 → expanded ↔ collapsed 토글
- [ ] 별점 ★ 탭 선택 → 평가 등록 시 rating 올바르게 전송
- [ ] 편집 모드: 모바일에서 지도 탭 → 새 기기 등록 모달 동작
- [ ] READ_ONLY 모드: 평가 폼 숨김 유지
